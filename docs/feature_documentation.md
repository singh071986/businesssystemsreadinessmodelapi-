# Feature Documentation
## Business Systems Readiness — ML Classification Engine

---

## 1. Overview

The classification engine accepts 12 question responses (A–D) and produces a
three-class prediction: **Foundation**, **Growth**, or **Optimization**.

The model uses **Logistic Regression (One-vs-Rest)** from scikit-learn, trained
on 1,350 synthetic samples generated by a deterministic rule-based classifier
that encodes the Business Systems Pathways framework.

---

## 2. Answer Encoding

Each of the 12 answer options is mapped to an integer on a maturity scale:

| Answer | Label         | Encoded Value | Maturity Level              |
|--------|---------------|---------------|-----------------------------|
| A      | Foundation    | 1             | Most basic / not systematised |
| B      | Foundation/Growth | 2        | Transitional                 |
| C      | Growth        | 3             | Systematised, consistent     |
| D      | Optimization  | 4             | Automated, strategic, scaled |

| Feature               | Range | Description                                                  |
|-----------------------|-------|--------------------------------------------------------------|
| `foundation_count`    | 0–11  | Count of questions where the answer falls in the Foundation signal range (see §4 below). Q10 is excluded (neutral baseline). |
| `growth_count`        | 0–12  | Count of questions where the encoded value equals 3 (C = Growth level). |
| `optimization_count`  | 0–12  | Count of questions where the encoded value equals 4 (D = Optimisation level). |
| `critical_blocker_count` | 0–3 | Count of critical Foundation blockers present (see §5 below). |
| `total_score`         | 12–48 | Sum of all 12 encoded answer values. Higher = more mature.  |

## 4. Foundation Signal Definitions

A "Foundation signal" for a question is defined as an answer in the Foundation
maturity range for that specific question, as documented in the Business Systems
Pathways framework:

| Question | Foundation signal answers              | Rationale (from Framework PDF)         |
|----------|----------------------------------------|----------------------------------------|
| Q1       | A (1) or B (2)                         | A = Foundation; B = Foundation/Growth bridge |
| Q2       | A (1) or B (2)                         | A = Foundation; B = Foundation stable  |
| Q3       | A (1) or B (2)                         | A = Foundation; B = Foundation         |
| Q4       | A (1) or B (2)                         | A = Foundation; B = Foundation/Growth  |
| Q5       | A (1) or B (2)                         | A = Foundation; B = Foundation         |
| Q6       | A (1) only                             | B = Growth early; not Foundation       |
| Q7       | A (1) only                             | B = Growth early; not Foundation       |
| Q8       | A (1) only                             | A = Foundation/Growth; B = Growth      |
| Q9       | A (1) only                             | A = Foundation/Growth; B = Growth early |
| Q10      | *None* (neutral baseline)              | A = Neutral; not a Foundation signal   |
| Q11      | A (1) only                             | B = Growth early; not Foundation       |
| Q12      | A (1) only                             | B = Growth                             |

**Maximum possible Foundation signals: 11** (Q10 is always excluded).

---

## 5. Critical Foundation Blockers

The following conditions, if present, automatically **prevent** Optimization
classification and indicate Foundation-level gaps:

| Blocker                  | Condition         | Meaning                                      |
|--------------------------|-------------------|----------------------------------------------|
| Undefined primary offer  | Q1 = 1 (A)        | Systems cannot stabilise without offer clarity |
| No structured CRM        | Q4 ≤ 2 (A or B)   | Customer journey is not tracked systematically |
| No follow-up automation  | Q5 ≤ 2 (A or B)   | Leads are followed up manually or inconsistently |

---

## 6. Classification Rules

The model training data is generated using these deterministic rules, which
the logistic regression learns to replicate:

### Rule 1 — Foundation Pathway
**Condition:** `foundation_count >= 3`  
Any response pattern yielding 3 or more Foundation signals is classified as
Foundation, regardless of optimization signals.

### Rule 2 — Optimization Pathway
**Condition:** All of the following must be true:
- `foundation_count < 3` (no Foundation trigger)
- No critical Foundation blocker present (Q1 ≠ A, Q4 > B, Q5 > B)
- `optimization_count >= 4` (at least 4 D-level answers)
- `q4 >= 3` (CRM structured with consistent usage)
- `q11 >= 3` (regular reporting in place)

### Rule 3 — Growth Pathway (default)
**Condition:** Neither Rule 1 nor Rule 2 conditions are met.  
The majority of answers are at the B–C level. The business has moved beyond
the most basic systems but is not yet at full automation and optimization.

## 7. Model Architecture

| Property            | Value                                                |
|---------------------|------------------------------------------------------|
| Algorithm           | `sklearn.multiclass.OneVsRestClassifier`             |
| Base estimator      | `sklearn.linear_model.LogisticRegression`            |
| Solver              | `lbfgs`                                              |
| Regularisation      | L2, C=1.0                                            |
| Max iterations      | 1,000                                                |
| Feature scaling     | `StandardScaler` (zero mean, unit variance)          |
| Classes             | Foundation (0), Growth (1), Optimization (2)        |
| Training samples    | 1,350 (450 per class, balanced)                     |
| Test set accuracy   | ≥ 98% (on 20% hold-out)                             |
| 5-fold CV accuracy  | ≥ 97%                                               |

## 9. Confidence Score

The `confidence_score` in the output is the model's predicted probability for
the winning class:

```
confidence_score = max(P(Foundation), P(Growth), P(Optimization))
```

When the deterministic rule classifier and the logistic regression disagree,
the deterministic rule takes precedence (rule-based override), and the
confidence score is updated to reflect the ML model's probability for the
rule-selected class.
