# Business Systems Readiness — ML Classification Engine

A Machine Learning classification engine that analyses a 12-question business
systems readiness survey and predicts the appropriate strategic pathway
(**Foundation**, **Growth**, or **Optimization**) using logistic regression.

## 1. Project Overview

Users complete a 12-question survey covering the following business system
domains:

| # | Domain                    |
|---|---------------------------|
| 1 | Clarity & Offer System    |
| 2 | Presence System           |
| 3 | Lead Capture System       |
| 4 | Customer Journey / CRM    |
| 5 | Nurture & Follow-Up       |
| 6 | Delivery System           |
| 7 | Payments & Offers         |
| 8 | Reactivation & Outreach   |
| 9 | Reputation & Reviews      |
| 10 | AI Engagement & Conversion |
| 11 | Reporting & Improvement   |
| 12 | Lifetime Value & Retention |

Each question is answered with a single letter (**A**, **B**, **C**, or **D**),
where A represents the most basic maturity level and D the most advanced.

The engine classifies the response pattern into one of three pathways and
returns a structured JSON result including the pathway, confidence score,
reasoning, a personalised summary, priority actions, and warnings.

## 5. Step-by-Step Usage

### Step 1 — Generate Training Data

The training data generator creates 1,350 synthetic labelled samples (450 per
class) using the deterministic rule-based classifier from the Business Systems
Pathways framework.

```bash
python src/generate_training_data.py
```

**Output:** `data/training_data.json`

```
Generating training data...
Saved 1350 samples to data/training_data.json
  Foundation: 450 samples
  Growth: 450 samples
  Optimization: 450 samples
```

> This step runs automatically when you call `train_model.py` if the file
> doesn't already exist.

---

### Step 2 — Train and Save the Model

```bash
python src/train_model.py
```

This script:
1. Loads (or generates) `data/training_data.json`
2. Encodes features into a 17-dimensional vector per sample
3. Splits data 80/20 (train/test) with stratification
4. Applies `StandardScaler` for feature normalisation
5. Trains a `OneVsRestClassifier(LogisticRegression(...))` model
6. Evaluates accuracy on the hold-out test set
7. Runs 5-fold cross-validation
8. Saves the model bundle to `models/pathway_classifier.pkl`

**Expected output:**
```
============================================================

============================================================
Loaded 1350 training samples.

Test set accuracy: 0.9815 (98.15%)

    accuracy                           0.98       270

Model bundle saved to models/pathway_classifier.pkl
```

### Step 3 — Run the Classifier (CLI)

#### Option A: Pass JSON directly as a command-line argument

```bash
python src/classifier.py '{"q1":"C","q2":"B","q3":"C","q4":"C","q5":"B","q6":"C","q7":"C","q8":"B","q9":"C","q10":"A","q11":"C","q12":"B"}'
```

#### Option B: Pass a JSON file with a single request

```bash
python src/classifier.py --file path/to/my_request.json
```

Important:
- Real user payloads should contain only the `responses` object.
- Use `data/user_request_examples.json` for production-style request examples.

```bash

python tests/test_validation.py
python tests/prod_smoke_test.py
```

Runs 12 named test cases across all three pathways and verifies that the
predicted pathway matches the expected outcome.

- `tests/test_classifier.py`: pathway classification correctness.
- `tests/test_validation.py`: required keys, allowed answers, and extra-key rejection.
- `tests/prod_smoke_test.py`: saved model artifact load + prod-style inference checks.


## 6. Input Schema

```json
{
  "responses": {
    "q1":  "A|B|C|D",
    "q2":  "A|B|C|D",
    "q3":  "A|B|C|D",
    "q4":  "A|B|C|D",
    "q5":  "A|B|C|D",
    "q6":  "A|B|C|D",
    "q7":  "A|B|C|D",
    "q8":  "A|B|C|D",
    "q9":  "A|B|C|D",
    "q10": "A|B|C|D",
    "q11": "A|B|C|D",
    "q12": "A|B|C|D"
  }
}
```

**Rules:**
- All 12 questions (`q1`–`q12`) are required.
- Only `q1` through `q12` are allowed (no extra keys).
- Answer values are case-insensitive (`a` and `A` are both valid).
- Each answer value must be a string and one of `A`, `B`, `C`, `D`.
- The outer `"responses"` wrapper is optional — bare dicts are also accepted.
- Invalid or missing questions raise a `ValueError` with a clear message.

## 10. Pathway Definitions

### Foundation
- **Trigger:** 3 or more Foundation-level signals detected
- **Characteristics:** Core systems are undefined or manual; offer, CRM, and follow-up automation are absent or inconsistent
- **Focus:** Build the essentials — offer clarity, website, CRM, lead capture, and basic follow-up automation

### Growth
- **Trigger:** Foundation signals < 3 and Optimization conditions not fully met
- **Characteristics:** Core systems are in place and running; delivery, payments, and reactivation need systematisation
- **Focus:** Standardise delivery, automate payments, activate database, build reputation systems

### Optimization
- **Trigger:** No Foundation blockers + 4+ Optimization (D) responses + strong CRM (Q4 ≥ C) + strong reporting (Q11 ≥ C)
- **Characteristics:** All systems are running; AI, lifecycle automation, and attribution are priority investments
- **Focus:** AI engagement, lifecycle-based automation, retention expansion, continuous improvement loops

## 11. Model Details

| Property              | Value                                                   |
|-----------------------|---------------------------------------------------------|
| Algorithm             | Logistic Regression (One-vs-Rest)                      |
| scikit-learn class    | `OneVsRestClassifier(LogisticRegression(...))`         |
| Solver                | `lbfgs`                                                 |
| Regularisation        | L2, C=1.0                                               |
| Features              | 17 (12 raw + 5 derived)                                 |
| Feature scaling       | StandardScaler                                          |
| Training samples      | 1,350 (450 per class, balanced)                        |
| Train/test split      | 80% / 20% (stratified)                                  |
| Test set accuracy     | **98.15%**                                              |
| 5-fold CV accuracy    | **97.26% ± 0.95%**                                      |
| Model file            | `models/pathway_classifier.pkl`                         |

For full feature documentation including Foundation signal definitions and
classification rule logic, see [docs/feature_documentation.md](docs/feature_documentation.md).
